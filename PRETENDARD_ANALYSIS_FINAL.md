- **챔퍼**: 없음

- **좌우 대칭**: 거의 대칭 (완벽하진 않음)
- **꼭짓점**: 날카로운 점 아님, blunt apex (약간 잘림)
- **ㅊ 상단 획**: 대각선 다리와 동일 두께

## 🎯 수정된 조화 전략

### 원칙
**Pretendard = 본문/UI 기반층**  
**축 = 브랜드/아이콘 강조층**

두 시스템을 **같은 단어에 섞지 않음** — 명확한 역할 분리로 충돌을 위계로 전환

---

### ① 점(·)

| 문맥 | 사용 시스템 | 형태 |
|---|---|---|
| Pretendard 본문 가운뎃점 | Pretendard | 완벽한 원 |
| 축 심볼 정점 | 축 | 작은 팔각형 (R6) |

**금지**: 같은 줄에 둘 다 사용

---

### ② 선(ㅡ·ㅣ)

| 문맥 | 사용 시스템 | 특징 |
|---|---|---|
| Pretendard 본문 | Pretendard | monolinear (1:1), 직각 cut |
| 축 아이콘/심볼 | 축 | ㅣ = 1.15× ㅡ, 45° 챔퍼 |
| UI 버튼 테두리 | 축 | 챔퍼 적용 (Pretendard 텍스트와 분리) |

**허용**: 타이틀/헤더에서 축 챔퍼 글리프를 display type으로 사용 (본문과 크기로 분리)

---

### ③ 삼각형(ㅅ·ㅈ·ㅊ) — **각도 충돌**

**Pretendard 53° vs 축 45° = 8° 차이**

| 요소 | 전략 |
|---|---|
| 본문 "천재", "축제" 등 | Pretendard 그대로 (53° 대각선) |
| 브랜드 워드 "축(Chuk)" | 축 팔각 ㅊ으로 치환 (타이틀/로고만) |
| UI 화살표, 아이콘 | 축 45° 엄격 준수 |

**예시**:
```
본문:  이런 천(Pretendard)재가  ← 53° ㅊ
타이틀: 축(축 팔각) Design System  ← 45° ㅊ
```

---

### ④ 사각형(ㅁ·ㅂ·ㄷ) — **꺾임부 충돌**

**Pretendard 직각 라운드 vs 축 45° 챔퍼**

| 요소 | 전략 |
|---|---|
| 본문 단어 | Pretendard (직각 꺾임, R≈0–1px) |
| UI 버튼/카드 프레임 | 축 챔퍼 (CSS `clip-path` or SVG) |
| 사각 아이콘 | 축 팔각 기반 |

**UI 구현**:
- Pretendard 텍스트 버튼: `border-radius: 2–4px` (작은 라운드)
- 축 브랜드 버튼: 4모서리 45° 챔퍼

---

### ⑤ 원(ㅇ) — **최대 충돌**

**Pretendard 가로 타원 (1.14:1, 슈퍼타원) vs 축 정팔각형 (1:1, 8개 직선 변)**

| 문맥 | 사용 시스템 | 처리 |
|---|---|---|
| 본문 전체 | Pretendard | 가로 타원 ㅇ |
| 브랜드 타이틀 | 축 (선택적) | 팔각 ㅇ |
| 아이콘/버튼 | 축 | 팔각 ㅇ |
| 로딩 스피너 | 축 | 회전하는 팔각 ㅇ |

**혼합 레이아웃 규칙**:
```
[헤더: Pretendard "공지사항"]  ← 가로 타원 ㅇ
↓
[본문: Pretendard 단락]
↓
[버튼: Pretendard "확인" + 축 팔각 프레임]  ← 텍스트는 타원, 테두리는 팔각
```

**타협안** — "공" 글자를 브랜드 타이틀에서만 축 팔각 ㅇ으로:
```
타이틀: 공(축 팔각)지사항  ← Display, 44px+
본문:   공(Pretendard)지사항을 확인하세요  ← Body, 16px
```

## 📐 Core Symbol "축 · Axis" 사용 전략

축 심볼(8방향 방사 + 중심점 + 굵은 세로축)은 **Pretendard 텍스트와 절대 겹치지 않음**

### 사용 위치
- 로고 (단독 또는 Pretendard 워드마크와 나란히)
- 섹션 구분선 장식
- 로딩/대기 상태 애니메이션
- 네비게이션 "홈" 아이콘

### 금지
- 본문 단락 inline 삽입 ✗
- Pretendard 글자와 같은 baseline 배치 ✗
