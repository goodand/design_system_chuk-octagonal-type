# Pretendard × 축(Chuk) — 시각 전략 (Visual Strategy)

## 🎯 전략의 출발점

두 시스템은 **DNA가 정반대**다. 이걸 억지로 섞으면 둘 다 죽는다.  
**충돌을 해결하지 말고, 충돌을 자산으로 만든다.**

---

## 💡 전략의 핵심 개념: **"중심과 궤도(Core & Orbit)"**

> 축(Chuk)은 **중심**, Pretendard는 **궤도**.  
> 축은 정지된 좌표계, Pretendard는 그 위를 흐르는 흐름.

```
        [축]
         ●
    ────┼────
         ●
    Pretendard
   (본문이 흐름)
```

**축**은 페이지에서 가장 강력하지만, 가장 적게 등장한다 — 로고, 섹션 시작점, 강조 모먼트.  
**Pretendard**는 가장 많이 등장하지만, 무대 위 조연이다 — 본문, UI, 라벨.

이 위계로 두 시스템의 충돌이 **계층 구조**로 변환된다.

---

## 📐 5가지 시각 전략 원칙

### ① **위계 분리 (Hierarchy Separation)**
같은 평면, 같은 크기에서 두 시스템을 만나게 하지 않는다.

```
축은 항상 더 크거나, 더 작거나, 다른 레이어에 있다.
같은 줄, 같은 px 크기로 인접 배치 → 절대 금지
```

| 레이어 | 시스템 | 크기 |
|---|---|---|
| Display (브랜드 모먼트) | 축 | 64px+ |
| Heading (섹션 타이틀) | Pretendard Bold | 32–48px |
| Body (본문) | Pretendard Regular | 14–18px |
| Icon (강조) | 축 | 24–48px (텍스트와 분리된 박스) |

---

### ② **공간 격리 (Spatial Quarantine)**
축이 등장하는 순간, 주변에 **숨 쉬는 공간**을 둔다.

- 축 글리프/심볼 주변 **최소 padding = 1×글리프 크기**
- 축과 Pretendard 사이 **명확한 시각적 구분선** (gap, divider, background block)

```
[Pretendard 본문]
                          ← 32px gap
   ┌─────────────┐
   │   [축 심볼]  │       ← 박스로 격리
   └─────────────┘
                          ← 32px gap
[Pretendard 본문 계속]
```

---

### ③ **충돌 흡수 (Conflict Absorption)**
충돌이 가장 심한 지점은 **건너뛴다 ≠ 강조한다**.

| 충돌 지점 | 처리 |
|---|---|
| **ㅇ (타원 vs 팔각)** | 본문 = Pretendard 100%, 브랜드 모먼트 = 축 100%. 같은 단어 혼용 ✗ |
| **종단 (직각 vs 챔퍼)** | UI 컨테이너만 챔퍼 (텍스트는 건드리지 않음) |
| **대각선 (53° vs 45°)** | 본문에서 축 글리프를 inline으로 쓰지 않음 |

**철칙**: 두 시스템의 차이가 보이는 곳에서는 **하나만 보이게 한다.**

---

### ④ **컬러 코딩 (Color Coding)**
축은 **하나의 색**에만 등장한다. Pretendard는 **모든 색**으로 자유롭게.

- **축 = 단일 강조색** (예: ink black, brand red, 또는 cream 위 ink)
- **Pretendard = 회색조 스펙트럼** + 본문 black

이 규칙으로 사용자는 **"색이 바뀌는 순간 = 시스템이 바뀌는 순간"** 을 학습한다.

```
"공지사항을 확인하세요"      ← 본문 그레이/블랙 (Pretendard)
       [축]                   ← 강조 레드 (축)
"오늘의 핵심"                 ← Pretendard 다시 블랙
```

---

### ⑤ **리듬 통제 (Rhythm Control)**
축은 **희소하게 등장**한다. 그래서 등장할 때마다 강력하다.

| 빈도 | 사용 예 |
|---|---|
| **1회 / 페이지** | 로고 |
| **2–3회 / 페이지** | 섹션 시작 마커, 강조 아이콘 |
| **0–1회 / 단락** | 절대 본문 inline 금지 |

축이 페이지에 5번 이상 등장하면 → **이미 너무 많다.**

---

## 🎨 4가지 시각 패턴 (Visual Patterns)

실제 화면에서 두 시스템이 만나는 4가지 정형 패턴.

---

### Pattern A — **Logo Lockup (로고 결합)**
> 축 심볼 + Pretendard 워드마크

```
●  Pretendard Co.
│
●
```

- 축 심볼 = 정사각 비율, 워드마크 좌측
- Pretendard = Bold 600+, 워드마크
- **수직 정렬**: 축 심볼 중앙선 = Pretendard cap-height 중앙
- **간격**: 축 너비의 30–40%

---

### Pattern B — **Section Anchor (섹션 앵커)**
> 큰 축 글리프가 섹션 시작을 표시, 그 아래 Pretendard 본문

```
┌─────┐
│  공  │  ← 축 팔각 ㅇ, 120px
└─────┘

공지사항입니다. 본문 내용은    ← Pretendard 16px
Pretendard로 흐릅니다.
```

- 축 글리프 = display size (64px+)
- Pretendard = body size (14–18px)
- 두 시스템이 **다른 크기 레이어**에 있어 충돌 안 함

---

### Pattern C — **Card with Chamfer Frame (챔퍼 프레임 카드)**
> Pretendard 텍스트를 축 챔퍼 프레임으로 감싸기

```
   ╱─────────────╲
  ╱               ╲
 │  공지사항       │   ← Pretendard
 │  Pretendard    │
  ╲               ╱
   ╲─────────────╱
```

- 카드 외곽 = 4모서리 45° 챔퍼 (`clip-path`)
- 내부 텍스트 = Pretendard (충돌 없음, 텍스트는 건드리지 않음)
- 챔퍼는 **컨테이너 레벨**에서만 적용

---

### Pattern D — **Display Headline (브랜드 헤드라인)**
> 한 단어만 축 글리프로, 나머지는 Pretendard

```
[축]은 새로운 시작입니다
 ↑
 축 팔각 (브랜드 단어)
```

- "축"이 들어간 **딱 한 글자만** 축 시스템
- 나머지 모든 텍스트 = Pretendard
- 축 글자는 **Pretendard 글자보다 1.2–1.5× 큰 cap-height**으로 강조

---

## 📊 사용 빈도 매트릭스

| 화면 종류 | Pretendard 비중 | 축 비중 | 패턴 |
|---|---|---|---|
| **랜딩 히어로** | 70% | 30% | A + D |
| **본문 페이지** | 95% | 5% | B (섹션 앵커만) |
| **UI 대시보드** | 90% | 10% | C (카드 프레임) |
| **로고 단독** | 50% | 50% | A |
| **아이콘 시스템** | 0% | 100% | 축 단독 |

---

## ⚖️ 의사결정 플로우차트

```
"이 자리에 무엇을 쓸까?"

이 텍스트가 본문인가?
├─ YES → Pretendard
└─ NO ↓

이 자리가 브랜드 모먼트인가?
├─ YES → 축 (단, 단일 단어 또는 심볼)
└─ NO ↓

이 자리가 UI 컨테이너인가?
├─ YES → Pretendard 텍스트 + 축 챔퍼 프레임
└─ NO ↓

이 자리가 아이콘/장식인가?
└─ YES → 축
```

## 🚫 금지 사항 (Anti-Patterns)

1. **축 글리프를 Pretendard 본문 중간에 inline 삽입** ✗
2. **같은 줄에 같은 크기로 두 시스템 나란히 배치** ✗
3. **축 시스템에 Pretendard의 픽셀 라운드 적용** ✗ (정체성 훼손)
4. **Pretendard에 축의 챔퍼 적용** ✗ (둘 다 어색)
5. **한 페이지에 축 5개 이상 등장** ✗ (희소성 손실)
6. **축의 ㅣ=1.15×ㅡ를 Pretendard에 강제** ✗ (Pretendard는 1.00)

## ✅ 핵심 요약

| 차원 | Pretendard | 축(Chuk) |
|---|---|---|
| **역할** | 흐름 (Flow) | 중심 (Core) |
| **빈도** | 95% | 5% |
| **목적** | 가독성, 정보 전달 | 브랜드, 강조, 정체성 |
| **레이어** | Body / UI | Display / Logo / Icon |
| **자유도** | 높음 (모든 색·크기) | 제한적 (특정 색·크기만) |

**한 줄 요약**:  
> **"Pretendard로 말하고, 축으로 외친다."**  
> Pretendard는 모든 문장을 담는 그릇, 축은 그 안에 던져진 단 하나의 보석.
